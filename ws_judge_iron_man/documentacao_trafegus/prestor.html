Documentação Webservice Prestor v0.9.3

Orientações gerais:

  • Essa documentação é uma extensão da documentação do Webservice Trafegus. Os métodos descritos aqui foram desenvolvidos para utilização do Prestor e do Trafegus Web.
  • Todos os envios e retornos deste webservice serão realizados em formato JSON.
  • O método de autenticação é o mesmo do Webservice Trafegus.
  • No geral os métodos tentam seguir o mesmo padrão do Webservice. Os métodos retornam sempre um json no formato: 
    {
    success: [], // informações no caso de sucesso
    error: [] // informações no caso de erro
}
    Os métodos também podem retornar uma chave "mensagem", com avisos para o usuário.

Changelog

  • v0.9.3
    • Adicionado tipo de macro no método POST /aparelho
  • v0.9.2
    • Adicionado o campo velocidade no método POST /posicao
  • v0.9.1
    • Removido campo "conta" do método POST /macro (o campo não é necessário)
    • O método GET /macro agora retorna também mensagens livres
    • Removido "tipo_macro" do retorno do método GET /macro
    • Se o usuário que estiver se cadastrando pelo webservice não for autônomo (empresa do usuário é o próprio usuário) será necessário que a GR faça a ativação dele, vinculando uma empresa e grupo de visualização
    • Adicionando campo "observacao" no cadastro dos usuários (para informar de qual empresa é o usuário)
    • Os campos "codigo_grupo_visualizacao" e "documento_empresa" do método POST /usuario agora são opcionais
    • No método POST /usuario, será criada uma empresa se o usuário for autônomo

/servidor [GET]

Retorna o link de conexão para o servidor do aplicativo de uma determinada empresa. O aplicativo utiliza esse método para escolher o servidor de acesso quando o usuário não deseja se conectar no ambiente em núvem

Autenticação

Não autenticado

Parâmetros

  • cnpj: documento da empresa
Exemplo de requisição
/servidor?cnpj=11222333444455

Resposta

  • link_conexao: URL para acesso ao servidor do aplicativo do cliente
Exemplo de resposta
{
    "link_conexao": "http://app.cliente.com/ws_rest/api"
}

/usuario [POST]

Esse método pode CRIAR ou ALTERAR um usuário.

Caso já exista um usuário com o login informado o serviço irá atualizar o usuário existente ao invés de criar um novo

Não é necessário estar autenticado para criar um novo usuário. Caso o usuário exista e não tenha sido informada autenticação ou o usuário autenticado não tenha permissão para alterar usuários, o retorno será um erro.

A pessoa do usuário pode ser informada por código ou objeto. Se for informado um código ou no caso do envio de objeto o webservice encontrar uma pessoa com o mesmo documento cadastrado no banco de dados, o cadastro existente será atualizado (respeitando as regras abaixo). Caso o código informado não seja encontrado será retornado um erro de validação.

Não é possível alterar a pessoa de um usuário já cadastrado nem informar pessoa que já está vinculada a outro usuário

Se a pessoa informada for do tipo "motorista" a pessoa será cadastrada como motorista automaticamente.

Se for informado um código de perfil esse perfil será vinculado, caso contrário será criado um novo perfil (e uma nova configuração) para usuário com base no perfil padrão do Prestor

Se o usuário for autônomo o documento da empresa deve ser o próprio documento da pessoa. Nesse caso será verificado se o documento ainda não existe da base e será criada uma empresa transportadora para o usuário

Parâmetros

  • login: Documento do usuário
  • senha: Senha do usuário
  • documento_empresa: Documento da empresa do usuário
  • codigo_perfil: Código do perfil do usuário
  • codigo_grupo_visualizacao: Código do grupo de visualização
  • email: E-mail do usuário
  • tempoSessaoWeb: Tempo de expiração da sessão no Trafegus Web
  • observacao: Observação do usuário (informar em qual empresa deve ser vinculado)
  • pessoa: Dados do usuário. É possível informar o Código de uma pessoa já existente ou informar os dados para que o sistema crie uma nova pessoa para o usuário.
    • nome: Nome do usuário
    • telefone: Telefone do usuário
    • tipo: Tipo da pessoa do usuário: "emb", "mot", "tra", "seg" e "gri"
    • documento: Documento do usuário
    • status: Código de status da pessoa
    • inscricaoEstadual: Inscrição estadual (no caso de pessoa jurídica)
    • razaoSocial: Razão social (no caso de pessoa jurídica)
    • site: Site do usuário (no caso de pessoa jurídica)
    • rg: Número da identidade (no caso de pessoa física)
    • orgaoEmissor: Órgão emissor (no caso de pessoa física)
    • estadoEmissorRg: Código do estado emissor (no caso de pessoa física)
    • sexo: Gênero "M" ou "F" (no caso de pessoa física)
    • dataNascimento: Data de nascimento no formato d/m/Y (no caso de pessoa física)
    • naturalidade: Código da cidade de naturalidade (no caso de pessoa física)
    • nomePai: Nome do pai (no caso de pessoa física)
    • nomeMae: Nome da mãe (no caso de pessoa física)
    • logradouro: Código do logradouro
    • numero: Número do imóvel
    • complemento: Complemento do imóvel
    • planoGr: Código do plano de gerenciamento do usuário
  • eventos_supervisor: Eventos do Supervisor
    • codigo_evento: Código do evento
    • gestao_risco: "0" ou "1" se a getão de risco está ativa
    • gestao_logistica: "0" ou "1" se a getão de logistica está ativa
    • gestao_administrativa: "0" ou "1" se a getão administrativa está ativa
  • eventos_supervisor_web: Eventos do Supervisor
    • codigo_evento: Código do evento
    • gestao_risco: "0" ou "1" se a getão de risco está ativa
    • gestao_logistica: "0" ou "1" se a getão de logistica está ativa
    • gestao_administrativa: "0" ou "1" se a getão administrativa está ativa
  • eventos_operador: Eventos do Supervisor
    • codigo_evento: Código do evento
    • gestao_risco: "0" ou "1" se a getão de risco está ativa
    • gestao_logistica: "0" ou "1" se a getão de logistica está ativa
    • gestao_administrativa: "0" ou "1" se a getão administrativa está ativa
